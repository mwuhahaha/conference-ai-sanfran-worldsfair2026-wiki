# Nearly Headless Web

## Overview
The nearly headless web is a product architecture where most discovery, understanding, and action can be handled by agents through machine-operable surfaces, while selected moments still keep a human-visible interface for judgment, preference, review, or handoff.

## Significance
Fully headless APIs are not always enough. Some user choices remain visual, comparative, or preference-heavy. Liad Yosef's agentic-web framing argues that the web is moving toward headless operation without becoming completely headless.

## Applied Use
- Expose structured actions for routine work.
- Preserve human UI for ambiguous, high-risk, or taste-based decisions.
- Make handoff explicit: agent prepares, human reviews, agent executes or records.
- Pair MCP Apps, WebMCP, browser automation, and normal web UI based on the job.
