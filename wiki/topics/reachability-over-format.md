# Reachability Over Format

## What It Is
Reachability over format is the idea that agent-readiness depends less on inventing the perfect agent-facing file format and more on making the right guidance reachable from the surfaces agents already trust: homepages, docs, linked resources, and visible product paths.

## Why It Matters
The clever move in [[2026-06-29-liad-yosef-rebuilding-the-web-for-agents|Rebuilding the web for agents]] is that agent-readiness is not only a protocol-design question. If an agent never discovers the file, tool, catalog, or policy, the format does not matter. The practical hack is to link agent-facing resources where agents naturally look.

## How To Use It
- Put agent guidance in or near public docs, product pages, and onboarding surfaces.
- Link machine-facing resources from human-readable pages instead of hiding them in isolated files.
- Treat navigation, naming, and source trust as part of the agent interface.
- Test with real agents, not only validators.
